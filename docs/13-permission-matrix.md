# 权限矩阵

## 1. 权限模型

权限由三层共同决定：

1. 租户角色。
2. 设备授权。
3. 资源状态。

租户角色：

- `owner`
- `admin`
- `approver`
- `viewer`

设备授权：

- `view`
- `approve`
- `admin`

系统主体：

- `device`: Agent 设备身份。
- `system`: Worker、超时任务。
- `policy`: 策略引擎。

## 2. 基本原则

- `owner` 拥有租户内全部权限。
- `admin` 拥有租户内管理权限，但不能删除 owner。
- `approver` 只能处理自己有权限的设备审批。
- `viewer` 只能查看，不允许提交审批决策。
- 设备令牌只能上报本设备数据，不能读取用户数据。
- Agent 本地 UI 提交审批动作时使用用户身份，不使用设备身份。
- 移动端查看会话必须通过租户成员关系和设备授权校验。

## 3. 租户级权限

| 操作 | owner | admin | approver | viewer |
|---|---:|---:|---:|---:|
| 查看租户信息 | 是 | 是 | 是 | 是 |
| 修改租户设置 | 是 | 是 | 否 | 否 |
| 管理成员 | 是 | 是 | 否 | 否 |
| 修改 owner | 是 | 否 | 否 | 否 |
| 查看审计日志 | 是 | 是 | 否 | 可选 |
| 导出审计日志 | 是 | 是 | 否 | 否 |
| 管理策略 | 是 | 是 | 否 | 否 |
| 查看策略 | 是 | 是 | 可选 | 可选 |

## 4. 设备权限

| 操作 | owner | admin | approver | viewer | device |
|---|---:|---:|---:|---:|---:|
| 创建设备激活码 | 是 | 是 | 否 | 否 | 否 |
| 绑定设备为 owner | 是 | 是 | 是 | 否 | 否 |
| 查看设备列表 | 是 | 是 | 授权设备 | 授权设备 | 否 |
| 查看设备详情 | 是 | 是 | 授权设备 | 授权设备 | 本设备有限 |
| 修改设备名称 | 是 | 是 | 设备 admin 授权 | 否 | 否 |
| 禁用设备 | 是 | 是 | 否 | 否 | 否 |
| 轮换设备令牌 | 是 | 是 | 设备 admin 授权 | 否 | 本设备可请求 |
| 管理设备授权 | 是 | 是 | 设备 admin 授权 | 否 | 否 |
| 上报设备心跳 | 否 | 否 | 否 | 否 | 本设备 |
| 上报会话和审批 | 否 | 否 | 否 | 否 | 本设备 |

说明：

- “授权设备”表示用户必须有 `device_grants` 或通过租户角色获得权限。
- 设备 owner 默认拥有该设备 `admin` 权限。

## 6. 审批权限

| 操作 | owner | admin | approver | viewer | device | system | policy |
|---|---:|---:|---:|---:|---:|---:|---:|
| 查看审批列表 | 是 | 是 | 授权设备 | 授权设备只读 | 本设备有限 | 否 | 否 |
| 查看审批详情 | 是 | 是 | 授权设备 | 授权设备只读 | 本设备有限 | 否 | 否 |
| 批准审批 | 是 | 是 | 授权设备 approve | 否 | 否 | 否 | 白名单策略 |
| 拒绝审批 | 是 | 是 | 授权设备 approve | 否 | 否 | 超时拒绝 | 自动拒绝 |
| 自定义回复 | 是 | 是 | 授权设备 approve | 否 | 否 | 否 | 否 |
| 标记本地接管 | 否 | 否 | 否 | 否 | 本设备 | 否 | 否 |
| ACK 投递结果 | 否 | 否 | 否 | 否 | 本设备 | 否 | 否 |
| 重试投递 | 是 | 是 | 否 | 否 | 否 | 是 | 否 |

审批动作要求：

- 用户动作必须带 `client_instance_id`。
- Agent 本地 UI 处理审批时，actor 是用户，client_type 是 `agent_desktop`。
- Agent 设备通道只能 ACK 回写结果，不能代表用户批准或拒绝。

| 操作 | owner | admin | 设备 owner | 设备 admin 授权 | approver | viewer |
|---|---:|---:|---:|---:|---:|---:|
| 查看授权列表 | 是 | 是 | 是 | 是 | 否 | 否 |
| 授权 view | 是 | 是 | 是 | 是 | 否 | 否 |
| 授权 approve | 是 | 是 | 是 | 是 | 否 | 否 |
| 授权 admin | 是 | 是 | 是 | 否 | 否 | 否 |

约束：

- 不能通过设备授权授予超过自己拥有的权限。
- 授权变更必须写审计。

## 9. 策略权限

| 操作 | owner | admin | approver | viewer |
|---|---:|---:|---:|---:|
| 查看策略 | 是 | 是 | 可选 | 可选 |
| 创建自动拒绝策略 | 是 | 是 | 否 | 否 |
| 创建自动批准策略 | 是 | 是 | 否 | 否 |
| 启用/禁用策略 | 是 | 是 | 否 | 否 |
| 删除策略 | 是 | 是 | 否 | 否 |

自动批准限制：

- 默认关闭。
- 高风险审批不允许全局自动批准。
- 自动批准策略必须有过期时间和审计备注。

## 10. 权限失败行为

- 未登录返回 `401 unauthorized`。
- 已登录但无租户权限返回 `403 forbidden`。
- 无权访问具体资源时，可以返回 `404` 隐藏资源存在性。
- 审批权限不足返回 `403 role_insufficient` 或 `device_access_denied`。
- viewer 提交审批返回 `403 role_insufficient`。

## 11. 权限检查顺序

推荐顺序：

1. 校验认证主体。
2. 校验租户成员关系。
3. 校验租户角色。
4. 校验设备归属。
5. 校验设备授权。
6. 校验资源状态。
7. 执行业务动作。

审计应记录权限拒绝事件，但避免泄露用户无权查看的敏感资源详情。
