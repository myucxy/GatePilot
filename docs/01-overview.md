# 项目概述

## 1. 建设目标

GatePilot 面向 AI Coding CLI 的远程审批与会话协同场景，解决用户不在电脑前、审批入口分散、操作缺少审计、企业环境需要统一治理的问题。

系统由四类端组成：

- 服务端: 负责账号、租户、设备、会话、审批、策略、审计、通知和消息路由。
- Web 管理端: 独立前端，负责日常审批、设备管理、策略配置、审计查询和运维视图。
- 移动端: 负责审批通知、快速处理、历史查看和设备状态提醒。
- 跨平台 Agent: 运行在 Windows/macOS/Linux 本机，负责启动或托管 AI CLI、采集终端输出、识别审批事件、回写审批结果。
- Agent 本地 UI: Agent 附带的本机交互入口，支持用户登录、查看本机会话、接收审批提醒和提交审批动作。

## 2. 角色定义

- 普通用户: 管理自己的设备和会话，处理自己的审批请求。
- 组织管理员: 管理组织成员、设备、策略、审计和系统配置。
- 系统管理员: 管理全局配置、OIDC、网关、部署和运维。
- Agent 设备: 已绑定到用户或组织的本地机器。
- 系统任务: 超时处理、自动拒绝、策略命中、投递重试等非人工行为。

## 3. 技术路线

- 服务端使用 Go 实现，MVP 可采用模块化单体，内部按 API、Realtime Gateway、Worker、Repository、Policy Engine 分层。
- Web 管理端使用独立前端，默认 React + TypeScript + Vite，通过 HTTP API 和 WebSocket 与服务端通信。
- Agent 使用 Go 实现跨平台命令执行和 PTY 接管。
- Windows 使用 ConPTY，macOS/Linux 使用 POSIX PTY。
- PostgreSQL 保存强一致业务数据，Redis 保存在线状态、短期锁、幂等缓存和轻量消息流。
- 推送通道仅作为提醒，审批详情、审批动作和投递结果必须通过服务端可靠接口完成。

## 4. 业务范围

### 4.1 第一版必须包含

- OIDC 登录接入。
- 用户、租户、成员关系的基础模型。
- 多客户端实例模型，支持同一用户多手机、多浏览器、Agent 本地 UI 同时在线。
- 设备注册、绑定、禁用、令牌轮换和心跳。
- 设备访问授权，支持一个账号管理多台机器，也支持一台机器授权给多个用户。
- Agent 与服务端的 WebSocket 长连接。
- Agent 启动或托管多个 AI CLI 会话，包括 Codex、Claude Code、openCode、GitHub Copilot CLI、Gemini CLI 和自定义 CLI。
- 审批事件识别、上报、去重和创建审批单。
- Web 管理端审批收件箱、审批详情、设备列表、会话列表。
- 移动端通过服务端查看有权限设备的会话列表和会话详情。
- 审批批准、拒绝、自定义回复、超时拒绝。
- Web、移动端、Agent 本地 UI 任意一端处理审批后，其他端实时同步处理结果。
- 审批决策下发、Agent 回写、投递确认和失败重试。
- 审计日志、基础查询和导出。
- Docker Compose 本地部署。

### 4.2 第一版暂不包含

- 复杂组织层级审批流。
- 计费系统。
- 云端运行 AI CLI。
- 浏览器内完整实时终端控制。
- 大规模 Kafka 集群和多地域部署。
- 高级自动批准策略。

## 5. MVP 收敛范围

第一版不要同时追求全部终端和全部规模能力。建议交付顺序：

1. Windows Agent + Go Server + Web 管理端完成端到端闭环。
2. 加入 macOS/Linux Agent，统一 PTY 抽象和适配器契约。
3. 加入 Flutter 移动端和 Push 提醒。
4. 拆分 Realtime Gateway 与 Worker，进入水平扩展形态。

MVP 的策略能力只允许：

- 手动审批。
- 超时自动拒绝。
- 低风险事件的显式白名单自动处理。

自动批准必须默认关闭，并需要组织管理员显式启用。

## 6. 非功能目标

- 可用性: 单实例故障不丢审批状态，Agent 断线后可补发和补收。
- 可追踪: 每个审批请求、审批动作、投递动作、回写结果必须有 `trace_id`。
- 可扩展: API、Gateway、Worker 可独立扩容。
- 安全: 设备令牌只保存哈希，终端输出默认脱敏，敏感内容最小化入库。
- 低耦合: Web、移动端、Agent 均只连接服务端，不互相直连。
- 可演进: 协议版本化，Agent 能与服务端协商能力集。

## 7. 关键成功指标

- 审批提醒触达延迟 P95 小于 3 秒。
- 在线 Agent 审批决策投递成功率大于 99.9%。
- 审批动作接口幂等成功率 100%。
- Agent 断线恢复后待投递消息不丢失。
- 单组织可管理多用户、多设备、多并发 CLI 会话。
- 数据模型支持按租户、时间和状态高效查询。
